# Codecov Setup Guide

## What is Codecov?

**Codecov** is a code coverage tracking service that measures what
percentage of your code is tested by your test suite. It:

- Integrates with your CI/CD pipeline (like GitHub Actions)
- Analyzes test coverage reports generated by `covr` (for R packages)
- Provides visual reports showing which lines of code are covered by
  tests
- Tracks coverage over time
- Shows coverage changes in pull requests

## Benefits

- **Identify untested code**: See which functions/lines aren’t covered
  by tests
- **Track progress**: Monitor how coverage improves over time
- **Quality assurance**: Ensure new code includes tests
- **Visual feedback**: Coverage reports in pull requests

## Setup Steps

### 1. Install Required Packages

Make sure you have `covr` installed (it’s used to generate coverage
reports):

``` r
install.packages("covr")
```

### 2. Add GitHub Actions Workflow

I’ve created a GitHub Actions workflow file
(`.github/workflows/test-coverage.yml`) that will: - Run your tests on
multiple R versions - Generate coverage reports using `covr` - Upload
coverage to Codecov

### 3. Connect to Codecov

1.  Go to [codecov.io](https://codecov.io) and sign up with GitHub
2.  Add your repository: `metapsy-project/metapsyTools`
3.  Codecov will automatically detect coverage reports from GitHub
    Actions

### 4. Add Badge to README

The badge has been added to your README. It will show: - Current test
coverage percentage - Link to detailed coverage report

The badge format is:

``` markdown
[![codecov](https://codecov.io/gh/metapsy-project/metapsyTools/branch/main/graph/badge.svg)](https://codecov.io/gh/metapsy-project/metapsyTools)
```

### 5. Generate Coverage Locally

You can test coverage locally before pushing:

``` r
# Install covr if not already installed
install.packages("covr")

# Generate coverage report
covr::package_coverage()

# View coverage report in browser
covr::report()
```

## How It Works

1.  **GitHub Actions runs**: When you push code or create a PR, GitHub
    Actions runs your tests
2.  **Coverage is generated**: `covr` analyzes which code is executed
    during tests
3.  **Report is uploaded**: Coverage data is sent to Codecov
4.  **Badge updates**: The badge in your README automatically updates
    with the latest coverage percentage
5.  **PR comments**: Codecov comments on PRs showing coverage changes

## Coverage Goals

- **Good**: 70-80% coverage
- **Excellent**: 80-90% coverage
- **Outstanding**: 90%+ coverage

Don’t aim for 100% - some code (like error handlers, edge cases) may be
difficult to test, and that’s okay!

## Troubleshooting

- **Badge not showing**: Make sure the workflow has run at least once
  and uploaded coverage
- **Coverage seems low**: Check which files have low coverage and add
  tests
- **Workflow fails**: Check GitHub Actions logs for errors
